# تشخیص اعداد دست‌نویس با XGBoost و رابط گرافیکی (Tkinter)

در این پروژه یک سیستم تشخیص اعداد دست‌نویس پیاده‌سازی شده است که شامل **ساخت دیتاست اختصاصی، ارزیابی کیفیت داده‌ها، مدلسازی با الگوریتم XGBoost و پیاده‌سازی رابط کاربری گرافیکی** می‌باشد.  
هدف اصلی پروژه، پیاده‌سازی یک pipeline کامل و قابل اتکا برای یک مسئله‌ی طبقه‌بندی تصویر بدون استفاده از یادگیری عمیق است.

---

## مراحل انجام پروژه

### 1️⃣ ساخت دیتاست
در مرحله‌ی اول، دیتاست خام تصاویر اعداد دست‌نویس پردازش و به یک دیتاست ساخت‌یافته تبدیل شد. اقدامات انجام‌شده در این بخش شامل موارد زیر است:

- تبدیل تصاویر به **سیاه‌وسفید (Grayscale)**
- ایجاد **پس‌زمینه سفید یکنواخت**
- تبدیل هر تصویر به آرایه‌ی عددی
- تخت‌سازی تصویر به بردار 784 عنصری (پیکسل 0 تا پیکسل 783)
- افزودن ستون `label` به‌عنوان برچسب کلاس

در نهایت دیتاستی ساخته شد که هر سطر آن نماینده‌ی یک تصویر و هر ستون آن یک ویژگی پیکسلی یا برچسب کلاس است.

---

### 2️⃣ بررسی کیفیت دیتاست
پس از ساخت دیتاست، برای اطمینان از صحت و کیفیت داده‌ها، مجموعه‌ای از تست‌های کنترلی روی دیتاست انجام شد. تمامی تست‌ها نتایج قابل قبول و منطقی داشتند.

تست‌های انجام‌شده عبارت‌اند از:

- **تست اول:** نمایش یک سطر دلخواه از دیتاست و بررسی تطابق تصویر و لیبل  
- **تست دوم:** نمایش چند سطر تصادفی برای بررسی صحت کلی داده‌ها  
- **تست سوم:** بررسی بازه‌ی مقادیر پیکسل‌ها  
- **تست چهارم:** بررسی بالانس بودن کلاس‌ها  
- **تست پنجم:** شناسایی تصاویر تقریباً خالی (با درصد بسیار بالای پیکسل‌های سفید)

نتایج این بررسی‌ها نشان داد دیتاست ساخته‌شده از کیفیت مناسبی برای مدلسازی برخوردار است.

---

### 3️⃣ مدلسازی
در این پروژه از الگوریتم **XGBoost** و کلاس `XGBClassifier` برای حل مسئله‌ی طبقه‌بندی استفاده شده است.

فرآیند ارزیابی مدل شامل مراحل زیر بوده است:

- آموزش مدل روی داده‌های آموزشی
- ارزیابی مدل با استفاده از:
  - `confusion_matrix`
  - `classification_report`
  - دقت مدل روی داده‌های آموزش و تست
- انجام تست‌های تکمیلی برای اطمینان از عدم بیش‌برازش (Overfitting):
  - **Cross-Validation**
  - **Sanity Check (Shuffle Label Test)**

نتایج این ارزیابی‌ها نشان داد مدل عملکرد بسیار مطلوبی داشته و یادگیری به‌درستی انجام شده است.

---

### 4️⃣ رابط کاربری گرافیکی (GUI)
برای نمایش عملی پروژه، یک رابط کاربری گرافیکی با استفاده از کتابخانه **Tkinter** پیاده‌سازی شده است.

قابلیت‌های رابط کاربری:
- آپلود تصویر دلخواه از عدد دست‌نویس
- پیش‌پردازش تصویر برای تطابق با دیتاست آموزشی
- پیش‌بینی عدد موجود در تصویر توسط مدل آموزش‌دیده
- نمایش نتیجه به کاربر به‌صورت گرافیکی

این بخش باعث شده پروژه از حالت صرفاً تحقیقاتی خارج شده و جنبه‌ی کاربردی پیدا کند.

---

## جمع‌بندی
در این پروژه یک pipeline کامل شامل **ساخت داده، ارزیابی کیفیت، مدلسازی، بررسی پایداری مدل و پیاده‌سازی رابط کاربری** پیاده‌سازی شده است.  
تمرکز اصلی پروژه بر روی اصول مهندسی یادگیری ماشین، صحت داده‌ها و ارزیابی علمی مدل بوده است، بدون استفاده از شبکه‌های عصبی عمیق.
